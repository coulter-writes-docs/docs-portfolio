## Project description

This is a technical writing portfolio built around WorkHop, a fictional B2B SaaS product for employee engagement, analytics, and reporting. The documentation I publish in this repository show a range of styles and content types.

>[!IMPORTANT]
> WorkHop is not a real product. The product features and API referenced in these documents do not exist. 

## Who this project is for

This project is intended for hiring managers, technical recruiters, or prospective collaborators who want to assess my technical writing range and quality across multiple documentation types.

## How to navigate this project

Each document in this repository is a standalone Markdown file. No build tools or local setup are needed; all files render directly on GitHub.

To read a document, click its link in the table above or browse the repository files directly. Use the toggle in the top left to switch between **Preview** and **Code** view.

## Tools and standards

I used the following tools and standards in this project:

* **Markdown** with some **HTML** for all content files
* **[The Good Docs Project](https://thegooddocsproject.dev/) templates** as a structural framework
* **The Good Docs Project style guide** for writing conventions, including plain language, sentence case, and information organisation
* **REST API conventions** for the API reference, following standard HTTP methods and response codes
* **GitHub** for hosting and sharing my work
* **Claude** for draft generation and some code formatting
* **Sublime Text** as a working code editor

## About the fictional product

WorkHop is a fictional B2B SaaS product for employee engagement, analytics, and reporting. It lets employees anonymously share feedback on their work experience and well-being with managers, directors, and HR teams.

The product, personas, data, and scenarios in this portfolio are for demonstration purposes only. Similarities to any real-world products or features is purely coincidental.
